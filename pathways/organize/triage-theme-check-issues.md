# Triage Theme Check Issues and PRs

**Function:** Organize  
**Type:** Task  
**Level:** Intermediate  

Help keep the Theme Check plugin moving by triaging open issues and pull requests. Confirm bugs, test PRs, categorize enhancement requests, and flag stale items. The Theme Check plugin is used by every theme reviewer to check themes against directory requirements, so keeping its issue tracker healthy directly supports the quality of the WordPress theme ecosystem.

## Before you start

Complete the [common setup](/handbook/pathways/before-you-begin/) first, then:

- **Complete:** You should be comfortable with WordPress themes, familiar with how the Theme Check plugin works, and able to read PHP code well enough to evaluate whether an issue is valid
- **Setup:** Set up [WordPress Playground](https://playground.wordpress.net/) and install the [Theme Check](https://wordpress.org/plugins/theme-check/) plugin for testing
- **Read:** Read through the [Themes triage](https://make.wordpress.org/themes/handbook/get-involved/themes-triage/) handbook page to understand the team's triage workflow, priorities, and how champions are assigned
- **Connect:** Join [#themes](https://wordpress.slack.com/archives/themes) on Slack and introduce yourself. Check the [meetings calendar](https://make.wordpress.org/meetings/#themes) for upcoming team meetings

## Steps

1. **Browse the open issues.** Go to the [Theme Check issues](https://github.com/WordPress/theme-check/issues) on GitHub and read through the open issues to get a sense of what's been reported and what's outstanding.
2. **Confirm or reproduce bugs.** Pick an issue that reports a bug. Install Theme Check in Playground, follow the steps described in the issue, and try to reproduce the problem. Comment on the issue with your findings, including your environment, what you tested, and whether you could reproduce it.
3. **Test open pull requests.** Check the [open PRs](https://github.com/WordPress/theme-check/pulls) and test whether the proposed changes work as intended and don't introduce new problems. Comment with your test results.
4. **Categorize and prioritize.** As you work through issues, note which ones are bugs, enhancement requests, duplicates, or no longer relevant. If an issue hasn't had activity in a long time, flag it as potentially stale. If you're unsure about priority, ask in #themes.
5. **Flag items for the team.** If you find issues that need a decision or a champion, bring them up in #themes or during a team meeting. The [triage sheet](https://docs.google.com/spreadsheets/d/1kmQ6YYe5Hq7d5c1COcuYFgXaIxuvLV4ZD1GsWsaH10w/) can be used as a reference for tracking.

## Contribution checklist

- Reviewed the open issues and PRs in the Theme Check repo
- Confirmed or tested at least one issue or PR with documented findings
- Commented on the issue or PR with your results
- Flagged stale or high-priority items for the team

## What happens next

A Themes team member or maintainer may respond to your comments, adjust labels, or assign a champion to the issue. Don't worry if your assessment gets corrected — that's part of learning the triage process.

As you get comfortable, attend team meetings to participate in triage discussions. Consistent triage work is one of the most valuable contributions you can make to keep the plugin moving forward.
