# Contribute to Playground

WordPress Playground lets anyone run WordPress instantly in the browser with no server required. Contributing here means improving the tools that power live demos, plugin previews, and interactive learning experiences across the WordPress ecosystem.

- **Prerequisite:** [Create a Playground Blueprint](https://make.wordpress.org/handbook/pathways/build/create-a-playground-blueprint/) — get familiar with how Playground works before diving into the codebase
- **Reference:** [Contributing to Playground](https://wordpress.github.io/wordpress-playground/contributing/code#contribute-pull-requests) — local development setup and PR workflow
- **Connect:** Introduce yourself in [#playground](https://wordpress.slack.com/archives/playground) on Slack. Attending meetings or reading meeting notes is also recommended.

## Steps

1. **Set up local development.** [Clone the playground repository](https://wordpress.github.io/wordpress-playground/contributing/code#contribute-pull-requests) and follow the instructions in its README.

2. **Learn the codebase.** Review the [architecture](https://wordpress.github.io/wordpress-playground/developers/architecture) and [coding principles](https://wordpress.github.io/wordpress-playground/contributing/coding-standards) to understand how the project is structured.

3. **Triage open issues.** Use the [triage guide](https://wordpress.github.io/wordpress-playground/contributing#triaging-issues) to help organize and prioritize issues.

4. **Claim an issue.** Choose an issue that isn't assigned yet and comment to let the team know you're working on it.

5. **Submit your pull request.** Test your solution, open a PR, and reference the issue number in your description.

## Contribution checklist

- Connected via Slack and reviewed recent meetings
- Commented on the issue to claim it
- Changes follow the project's coding standards
- PR references the related issue
- Tests pass locally

## What happens next

A maintainer will review your PR — typically within a few days. If it's been a week, a polite ping in #playground is fine.

When you're ready, pick up another issue, settle into a feature area, or review other contributors' PRs.

## Help

Stuck? Check the [getting help guide](https://make.wordpress.org/handbook/pathways/before-you-begin/#getting-help), then ask in [#playground](https://wordpress.slack.com/archives/playground).
