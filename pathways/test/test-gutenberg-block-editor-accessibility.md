# Test Gutenberg Block Editor Accessibility

Help make the WordPress block editor work for everyone. You'll test blocks and features in the Gutenberg editor with your keyboard and a screen reader, then report what you find so the team can fix accessibility barriers.

- **Reference:** [Gutenberg accessibility testing guide](https://make.wordpress.org/accessibility/gutenberg-testing/)
- **Connect:** Join [#accessibility](https://wordpress.slack.com/archives/accessibility) on Slack and introduce yourself

## Steps

1. **Open the test environment.** Launch the [Gutenberg Playground environment](https://playground.wordpress.net/?plugin=gutenberg&url=/wp-admin/post-new.php) to start testing the latest editor in your browser — no local setup required.

2. **Pick something to test.** Choose an open issue from the [Needs Accessibility Feedback](https://github.com/WordPress/gutenberg/issues?q=is%3Aopen+is%3Aissue+label%3A%22Needs+Accessibility+Feedback%22) list, or test a block or feature yourself. The Tuesday accessibility bug scrubs in #accessibility are a good place to find work and learn the ropes.

3. **Test the editor.** Work through common tasks: adding and editing paragraphs, headings, images, lists, tables, and embeds, using your keyboard and a screen reader. The [Gutenberg testing guide](https://make.wordpress.org/accessibility/gutenberg-testing/) has a sample script, and the [keyboard](https://wpaccessibility.org/docs/testing/keyboard/) and [screen reader](https://wpaccessibility.org/docs/testing/screen-readers/) guides show how. You don't need to fix anything, just note what doesn't work.

4. **Search before you report.** Check the [Gutenberg issues](https://github.com/WordPress/gutenberg/issues) for your problem first. If it's already filed, add a comment confirming it still happens.

5. **Report what you found.** Comment on the issue or open a new one, including the date, the Gutenberg version, your browser, your operating system, and the assistive technology you used.

## Contribution checklist

- Tested the Gutenberg editor with your keyboard and a screen reader
- Searched existing issues before reporting
- Reported with date, Gutenberg version, browser, OS, and assistive technology used

## What happens next

The Gutenberg team and accessibility contributors review issues on GitHub, often during the Tuesday bug scrubs, and decide what needs fixing. You don't need to write code or fix the issue yourself. If your report gets no response, raise it in #accessibility or bring it to a bug scrub.

Once you've reported one issue, pick up another from the Needs Accessibility Feedback list, or join a bug scrub to test alongside the team.
