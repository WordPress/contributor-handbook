# Create a Pathway Guide

Turn a pathway idea into a finished guide. Pick up an issue from the project board, draft it, get feedback, and submit a pull request to get published.

- **Reference:** [About Pathways](https://make.wordpress.org/handbook/pathways/about/)
- **Connect:** Join [#core-program](https://wordpress.slack.com/archives/core-program) and introduce yourself

## Steps

1. **Familiarize yourself with the project.** Review the [Contributor Handbook GitHub repository](https://github.com/WordPress/contributor-handbook) and the [pathway project board](https://github.com/orgs/WordPress/projects/282/views/1).

2. **Choose or create an issue.** Either pick an open issue from the [project board](https://github.com/orgs/WordPress/projects/282/views/1), or [suggest a new guide](https://make.wordpress.org/handbook/pathways/organize/suggest-a-pathway/) by filing an issue. If working on an existing issue, comment `/assign` to claim it.

3. **Research the pathway.** Explore using linked resources, handbooks, tutorials, [Learn WordPress](https://learn.wordpress.org/), [WordPress.tv](https://wordpress.tv/), and team Slack channels. If you already know this pathway well, note any outdated information you find.

4. **Draft the guide.** Use [PROMPT.md](https://github.com/WordPress/contributor-handbook/blob/main/pathways/PROMPT.md) and AI, or [TEMPLATE.md](https://github.com/WordPress/contributor-handbook/blob/main/pathways/TEMPLATE.md) as your guide. Post your draft (with markdown) as a comment on the issue.

5. **Test and refine.** Walk through your draft as a new contributor would. Get feedback from the relevant team's Slack channel and update based on what you learn.

6. **Submit for publication.** Comment `/done` on the issue to signal it's ready. If you're able, set its Projects > Status to Update Manifest and submit a pull request with your .md file in the correct directory. Otherwise, an admin can create the PR from your comment.

## Contribution checklist

- All links work and are current
- Draft is scannable and fits the TEMPLATE.md structure
- Pathway tested, feedback requested, and refined accordingly
- PR submitted referencing the issue, or guide shared as markdown in a comment

## What happens next

Maintainers will review your PR for clarity, accuracy, and completeness. Once approved, the pathway will be merged and published to the handbook.

When you're ready, pick up another pathway from the project board or help review someone else's draft. You can also [suggest new guides](https://github.com/WordPress/contributor-handbook/issues/new?template=2-new-pathway.yml).

## Help

Stuck? Check the [getting help guide](https://make.wordpress.org/handbook/pathways/before-you-begin/#getting-help), then ask in [#core-program](https://wordpress.slack.com/archives/core-program).
